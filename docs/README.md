# VISTA Documentation

This directory contains the detailed documentation for each part of the VISTA framework.

---

## The five dimensions

- [Value](value.md) - Did the work create meaningful user or business benefit?
- [Impact](impact.md) - Did it advance strategic goals and competitive position?
- [Safety](safety.md) - Did it avoid hidden downside, including security failures, and hold up in reality?
- [Timing](timing.md) - Was capacity directed to the right work at the right time?
- [Adaptation](adaptation.md) - Is the system learning and improving over time?

## How VISTA works

- [Decision Model](decision-model.md) - The three-layer model for choosing, shaping, and evaluating work
- [ODIM](odim.md) - Larry Maccherone's Outcome, Decision, Insight, Measure framework for designing metrics that connect to business results
- [Scoring Template](scoring.md) - Rubrics and forcing questions for before-coding, during-review, and after-release
- [Team Profiles](profiles.md) - Diagnostic patterns for recognizing where your team is and what to change
- [Dashboards](dashboards.md) - MVP dashboard designs for executives, delivery leaders, and teams

## Principles

## Positioning

VISTA complements delivery metrics frameworks like [DORA](https://en.wikipedia.org/wiki/DORA_metrics) ([dora.dev](https://dora.dev/)) and [SPACE](https://queue.acm.org/detail.cfm?id=3454124), from "The SPACE of Developer Productivity" by Nicole Forsgren, Margaret-Anne Storey, Chandra Maddila, Thomas Zimmermann, Brian Houck, and Jenna Butler. Those frameworks measure **flow** - how fast work moves through the system. VISTA measures **direction** - whether the movement was worth it. See the [Decision Model](decision-model.md) for details.

# Delivery Cost Is Broader Than Coding Cost

## Principle

**Delivery cost is broader than coding cost.**

The true cost of a work item includes more than implementation effort. It also includes review, verification, rollout, activation, coordination, support burden, cleanup, and opportunity cost.

## Why this matters

AI dramatically lowers the coding portion of delivery cost, which makes the *non-coding* portions relatively larger and more important.

A feature that takes one day to code with AI may still require:
- Two days of review across three reviewers
- A security review for data-handling changes
- Documentation updates and support team training
- A migration path for existing customers
- Sales enablement for the go-to-market team

If the team only measures coding time, this item looks cheap. If the team measures full delivery cost — including activation burden, review effort, and displaced alternatives — the real cost becomes visible.

This is especially important for items with high **activation burden**: the non-development work required to realize value. A feature with zero activation burden (deploy and done) is rare. Most features need cross-functional follow-through that engineering metrics never see.

---

Delivery is a full-system activity. Coding cost is the part that AI is making cheaper. Everything else stays the same or gets harder.
